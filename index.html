Agentic AI · LangGraph · RAG

Enterprise Financial Intelligence Platform

Business Problem

Financial analysts spend hours parsing dense reports, cross-referencing data sources, and synthesising insights — work that requires domain knowledge but follows predictable reasoning patterns. The question: can a governed AI system perform this reliably enough to trust in a regulated context?

  • Supervisor-Worker multi-agent hierarchy — LangGraph orchestrates Research, Quantitative Analysis, and Reporting agents in parallel, enforcing separation of concerns
  • Hybrid RAG pipeline — Keyword + Semantic retrieval via pgvector, purpose-built for high-accuracy extraction from complex financial documents
  • Time-travel session persistence — PostgreSQL checkpointers maintain stateful conversations across restarts with no re-ingestion
  • HITL interrupt governance — High-stakes actions (e.g. simulated trades) pause for human validation before execution — reusable pattern for any regulated workflow
  • Streaming generative UI — Next.js renders real-time interactive charts alongside agent responses as the analysis flows

02
Text-to-SQL · On-Premise LLM · Privacy-First

Self-Hosted Natural Language Analytics Platform

Business Problem

Enterprise data teams want "ask your data" capability, but cloud-based tools (ThoughtSpot, Databricks AI/BI Genie) require sending sensitive business data to external providers. For AEC firms, financial institutions, and healthcare organisations, that's a non-starter. This system delivers equivalent capability entirely on-premise.

  • On-premise LLM stack — Qwen 3B via Ollama, zero data leaves the environment; critical for AEC project financials, bid data, and client contracts
  • Self-correction loop — SQL execution failures feed the error + failed query back to the LLM for autonomous correction, retrying up to a configurable maximum
  • YAML semantic layer — Business metrics (formula, dimensions, synonyms) defined in config files, converted to rich LLM training context; non-technical users extend without touching code
  • SELECT-only guardrails — Pre-execution validation blocks all write operations (INSERT, UPDATE, DELETE, DROP) — safe to expose against production databases
  • In-session result cache — Normalized question hash keys serve repeated queries instantly; Redis-backed semantic routing for similar questions
